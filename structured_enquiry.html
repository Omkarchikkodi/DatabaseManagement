Structured Enquiry: Pharmacy Management System

Problem Statement

The Pharmacy Management System aims to address the complexities of managing pharmaceutical company contracts, drug inventories, sales tracking, prescriptions, and patient-doctor interactions. The primary goal is to create an efficient, secure, and scalable database system to handle these operations seamlessly.

Objectives

  • Design a database to manage relationships between Pharmaceutical Companies, Pharmacies, Doctors, Patients, and Drugs.
  • Ensure data integrity, security, and consistency across all operations.
  • Enable efficient querying for data retrieval and reporting purposes.
  • Provide support for transactional operations with ACID properties.

System Design

The system is designed using an Entity-Relationship (ER) model with the following key entities and relationships:

  • Entities: Pharmacies, Pharmaceutical Companies, Doctors, Patients, Drugs.
  • Relationships: Contracts (between Pharmacies and Companies), Prescriptions (between Doctors and Patients), Sales (between Pharmacies and Drugs).
  • Normalization: Database normalized to BCNF to ensure minimal redundancy.
  • Schema Design: Includes primary and foreign keys, along with constraints to maintain data integrity.

Implementation

The system was implemented using SQL for database creation and operations. Key aspects of the implementation include:

  • Created normalized tables with relationships and constraints.
  • Executed complex queries for sales tracking, contract management, and prescription analysis.
  • Designed views for simplified data retrieval and enhanced reporting capabilities.
  • Implemented transactional operations to ensure consistency in multi-user environments.

Reflection on Learning

This assignment significantly enhanced my understanding of database design principles and their practical applications in real-world scenarios. By working on the Pharmacy Management System, I gained insights into structuring data efficiently, securing sensitive information, and ensuring data consistency in multi-user environments.

The process also highlighted the importance of normalization, query optimization, and transaction management in building scalable and reliable database systems. This learning experience has strengthened my foundation in database management systems and prepared me to tackle more complex challenges in the future.
